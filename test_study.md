### 상속

##### 부모클래스에 만들어진 필드, 메소드를 자식클래스가 물려받음 <br>
부모의 생물학적 특성을 물려받는 유전과 유사<Br>
 -> 자식클래스 간결

#### 상속의 개념<br>
부모클래스의 멤버(필드, 메서드, 이너클래스)를 자식 클래스가 내려받아(상속) 클래스 내부에 포함

#### 자바 상속의 특징<br>
부모 클래스: super class / 자식 클래스: sub class<br>
 상속 횟수에 제한 없음<br>
 자바에서 최상위 클래스는 object클래스, 모든 클래스는 object 클래스를 상속받음

 #### object 클래스 특징
 모든 클래스의 상속 계층도의 최상위에 위치<br>
 다른 클래스를 상속 받지 않는 모든 클래스들은 자동으로 object 클래스를 상속받음<br>
 toString(),equals()와 같은 모든 인스턴스가 가져야 할 기본적인 11개의 메서드 정의
 
 ### 상속을 구현하는 경우
 상위 클래스는 하위 클래스보다 더 일반적인 개념과 기능을 가짐<br>
 하위 클래스는 상위 클래스 보다 더 구체적인 개념과 기능을 가짐<br>
 하위 클래스가 상위 클래스의 속성과 기능을 확장한다는 의미<br>
 class human **extends** MM
 
 ### 상속의 필요성
 기존의 검증된 class를 이용해서 빠르고 쉽게 새로운 class를 만들 수 있음
 
 ### 상속 개념의 활용
 #### 상속의 효과
 부모 클래스를 재사용해 자식 클래스 빨리 개발<br>
 반복된 코드 중복 줄임, 유지 보수 편리성, 객체 다형성 구현
 #### 상속 대상 제한
 부모 클래스의 private 접근 갖는 필드와 메소드 제외<br>
 부모 클래스가 다른 패키지에 있을 경우, default 접근 갖는 필드와 메소드도 제외
 
 ### 객체 지향에서 상속의 장점
 #### 클래스 간결화
 멤버의 중복 작성 불필요
 #### 클래스 관리 용이
 클래스들의 계층적 분류
 #### 소프트웨어의 생산성 향상
 클래스 재사용과 확장 용이 <br>
 새로운 클래스의 작성 속도 빠름
 
 ### 객체 간의 상속의 의미
 #### 클래스 상속
 새로운 클래스를 정의 할 때 이미 구현된 클래스를 상속받아서 속성이나 기능을 확장하여 클래스를 구현<br>
 이미 구현된 클래스보다 더 구체적인 기능을 가진 클래스를 구현해야 할때 기존 클래스를 상속함
 #### 상속하는 클래스: 
 상위 클래스, parent class, base class, super class
 #### 상속받는 클래스:
 하위 클래스, child class, derived class, subclass
 
 ### 클래스 상속의 선언
 #### 자바의 상속 선언
 class 클래스이름 **extends** 상위클래스이름 { 멤버변수선언, 생성자, 메소드선언}<br>
 부모클래스(super class), 자식클래스(sub class)<br>
 extends -> 슈퍼 클래스를 확장
 
 자식 클래스는 부모 클래스의 모든 자원의 사용할 수 있지만, private 접근자의 속성과 메서드는 사용할 수 없음
 
 ### 클래스 간의 관계 - is a, has a
 #### 상속관계
 **is a** -> 관계를 문장으로 만들어봤을 때 '~은 이다'라는 문장 성립, 상속 특징 따름<br>
 공통부분은 조상에서 관리하고 개별부분은 자손에서 관리<br>
 조상의 변경은 자손에 영향을 미치지만, 자손의 변경은 조상에 아무런 영향을 미치지 않는다.
  #### 포함관계
 **has a** -> 관계를 문장으로 만들어봤을 때 '~은 ~을 가지고 있다'라는 문장 성립<br>
 한 클래스의 멤버변수로 다른 클래스를 선언하며 사용<br>
 작은 단위의 클래스를 먼저 만들고, 이것들을 포함하여 하나의 커다란 클래스를 만듬<br>
 한 클래스의 멤버변수로 다른 클래스를 선언하는 것<br>
 작은 단위의 클래스를 먼저 만들고, 이 들을 조합해서 하나의 커다란 클래스를 만듬
 
 ### 클래스간의 관계결정하기 - 상속 vs 포함
 가능한 한 많은 관계를 맺어주어 재사용을 높이고 관리하기 쉽게 함
 
 ### Super 키워드
 #### 자식 클래스가 자신을 생성할 때 부모 클래스의 생성자를 불러 초기화 할 때 사용
 반면에 this는 자기 자신 클래스 의미, this() 자기 자신 클래스의 생성자 호출
 #### 상속관계에서 부모클래스의 필드나 메서드를 참조하기 위해 사용<br>
 (부모 클래스의 메서드나 필드를 자식 클래스에서 재정의 했을 경우, 부모클래스를 사용하기 위해 사용
 
  ### 상속과 생성자
 상속된 클래스로부터 객체가 생성될 때 상위 클래스의 생성자가 하위 클래스의 생성자보다 먼저 수행<br>
 상위 클래스의 묵시적인 생성자(매개변수x)수행<br>
 묵시적인 생성자가 아닌 다른 생성자를 호출(오버로딩의 경우)하고자 한다면 예약어 super를 사용하여 명시적으로 호출<br>
 super 문장은 반드시 첫번째 라인에 와야 한다.

 ### 오버로딩 vs 오버라이딩
 #### 오버로딩
 기존에 없는 새로운 메서드를 정의하는 것<br>
 1개의 클래스 내에서 동일한 매개변수를 다르게 정의하여 사용하는 것을 말함<br>
 외부로부터 다양한 입력값(매개값)을 받기 위해서 사용
 #### 오버라이딩
 상속받은 메서드의 내용을 변경하는 것

 ### 생성자 오버로딩
 클래스에 하나 이상의 생성자를 중첩하여 사용<br>
 여러 개의 생성자를 사용할 때는 생성자의 이름은 같지만 매개변수의 타입과 개수는 달라야 한다.
 
 ### 메서드 오버로딩
 생성자 오버로딩과 같은 개념<br>
 한 클래스 내에서 같은 이름의 메소드를 사용<br>
 같은 이름의 메소드는 매개변수의 타입과 개수가 달라야 한다.<br>
 다향성제공
 
 ### 오버라이딩
 조상클래스로부터 상속받은 메서드의 내용을 상속받는 클래스에 맞게 변경하는 것 <br>
 수퍼 클래스에 선언된 메소드를, 각 서브 클래스들이 자신만의 내용으로 새로 구현 <br> 
 상속을 통해 '하나의 인터페이스'에 서로 다른 내용 구현이라는 객체 지향의 다향성 실현<br>
 실행 시간 다형성 실형 -> 동적 바인딩
 
 ### 다향성
 같은 타입이지만 실행 결과가 다양한 객체 대입이 가능한 성질<br>
 부모 타입에는 모든 자식 객체가 대입 가능(자식 타입은 부모 타입으로 자동 타입 변환)<br>
 효과 : 객체 부품화 
 
 ### @Overrideing 어노테이션
어노테이션: 주석 <br> 컴파일러에게 특별한 정보를 제공해주는 역할<br> @Override: 재정의된 메서드 라는 의미로 선언부가 기존의 메서드와 다른 경우 오류가 남
 
 ### 오버라이딩의 조건
 #### 1.선언부가 같아야함 (이름, 매개변수, 리턴타입)
 #### 2. 접근제어자를 좁은 범위로 변경할 수 없음
 ##### 조상의 메서드가 protected라면, 범위가 같거나 넓은 protected나 public으로만 변경할 수 있음<br> (private < default < protected < public)
#### 3. 조상클래스의 메서드보다 많은 수의 예외를 선언할 수 없음
#### 4. 오버라이딩은 상속을 받은 후 할 수 있음
#### 5. @Override 어노테이션
##### 컴파일러에게  부모 클래스의 메서드 선언부와 동일한지 검사 지시 <br> 정확한 메서드 재정의를 위해 붙여주면 됨
 
 ### 메서드 오버라이딩
 #### 슈펴 클래스의 메서드를 서브 클래스에서 재정의
 ##### 슈퍼 클래스 메서드의 이름, 매개변수 타입 및 개수, 리턴 타입 등 모든 것 동일하게 작성
 #### 메서드 무시하기, 덮어쓰기로 번역 <br> 동적바인딩 발생
 ##### 서브 클래스에 오버라이딩된 메서드가 무조건 실행되는 동적 바인딩
